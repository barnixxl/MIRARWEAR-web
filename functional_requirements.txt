Функциональные требования к проекту Mirar Wear

1. Функция «Просмотр и фильтрация товаров»
• Описание: Система должна отображать список товаров с возможностью фильтрации по параметрам: категория, размер, цена, наличие. Доступен текстовый поиск по названию.
• Источник данных: База данных товаров (localStorage / база сайта).
• Действия: Пользователь выбирает параметры фильтрации и поиска, система отображает соответствующие товары. Фильтрация по нескольким категориям, размерам и диапазону цены.
• Формат вывода: Галерея товаров с изображениями, описанием, ценой, списком размеров и статусом (в наличии / нет в наличии).

2. Функция «Регистрация и авторизация пользователей»
• Описание: Система позволяет пользователю зарегистрироваться, войти и выйти из аккаунта.
• Источник данных: База пользователей (localStorage).
• Действия: Пользователь вводит имя, email, телефон, пароль; проходит регистрацию или выполняет вход по email/паролю. Система валидирует данные и сохраняет/авторизует пользователя.
• Формат вывода: Модальные окна для регистрации/входа, отображение информации о пользователе в меню после входа, доступен выход.

3. Функция «Оформление заказа»
• Описание: Пользователь может оформить заказ на выбранный товар с указанием размера, имени, телефона, email.
• Источник данных: База заказов, база товаров (localStorage).
• Действия: Оформление заказа через форму, система сохраняет заказ, уменьшает количество товара на складе, отображает результат (успех или сообщение об ошибке).
• Формат вывода: Модальное окно с формой заказа, подтверждение успешного создания заказа.

4. Функция «Личный кабинет пользователя — История заказов»
• Описание: Авторизованный пользователь может просматривать историю своих заказов и отменять их.
• Источник данных: База заказов (localStorage).
• Действия: Вкладка/секция истории заказов — выводит список заказов с данными: товар, изображение, цена, размер, статус, дата, кнопка отмены.
• Формат вывода: Таблица заказов.

5. Функция «Админ-панель»
• Описание: Система позволяет администратору управлять товарами, категориями, пользователями, заказами и другими администраторами.
• Источник данных: Базы товаров, категорий, пользователей, заказов, админов (localStorage).
• Действия: 
   — Просмотр статистики (пользователи, заказы, новые заказы, админы.)
   — Управление товарами (создание, редактирование, удаление, поиск и фильтрация по категориям, цене, количеству, размерам).
   — Управление категориями (создание, изменение, удаление категорий).
   — Управление пользователями (просмотр и удаление пользователей).
   — Управление заказами (просмотр, изменение статуса, удаление заказов, отправка письма клиенту).
   — Управление администраторами (создание, удаление, просмотр админов; только для глав.админа).
• Формат вывода: Админ-панель с вкладками, таблицы данных, модальные окна для создания/редактирования, отчёты и статистика.

6. Функция «Контактная информация и страница “О нас"»
• Описание: Отдельные страницы сайта с основными контактами и описанием бренда.
• Источник данных: Статические HTML страницы.
• Действия: Просмотр информации посетителями, переход по ссылкам на соцсети.
• Формат вывода: Разделы “О нас” и “Контакты” с текстом, медиа, ссылками.

7. Функция «Переключение темы (светлая/тёмная)”
• Описание: Пользователь может переключать тему интерфейса (тёмная/светлая).
• Источник данных: localStorage (сохраняется выбранная тема).
• Действия: Переключение по кнопке, смена стилей приложения.
• Формат вывода: Кнопка переключения темы, глобальная смена цветов интерфейса.

8. Функция «Импорт/экспорт данных (админ)»
• Описание: Админ может экспортировать все данные сайта (товары, категории, заказы, пользователи, админы) в файл JSON или SQL, а также импортировать их для миграции/резервного копирования.
• Источник данных: localStorage / база сайта.
• Действия: Экспорт данных одной кнопкой, формирование файла для резервного копирования, тестовый импорт данных через скрипт.
• Формат вывода: Скачивание файла, отчёт об успешном экспорте/импорте.

9. Функция «Импорт/экспорт данных (админ)»
• Описание: Админ может экспортировать все данные сайта (товары, категории, заказы, пользователи, админы) в файл JSON или SQL, а также импортировать их для миграции/резервного копирования.
• Источник данных: localStorage / база сайта.
• Действия: Экспорт данных одной кнопкой, формирование файла для резервного копирования, тестовый импорт данных через скрипт.
• Формат вывода: Скачивание файла, отчёт об успешном экспорте/импорте.

10. Функция «Безопасность доступа к административным действиям»
• Описание: Доступ к административным разделам и действиям возможен только для авторизованных админов, часть функций — только для главного админа.
• Источник данных: Авторизационные данные (localStorage).
• Действия: Проверка прав, блокировка неавторизованного доступа, отображение ошибок.
• Формат вывода: Сообщения об ошибках, редирект на главную при недостаточных правах.
